# hs-opentelemetry-exporter-otlp

[![hs-opentelemetry-exporter-otlp](https://img.shields.io/hackage/v/hs-opentelemetry-exporter-otlp?style=flat-square&logo=haskell&label=hs-opentelemetry-exporter-otlp&labelColor=5D4F85)](https://hackage.haskell.org/package/hs-opentelemetry-exporter-otlp)

OTLP exporter for traces, metrics, and logs. The OTLP wire types come from
[`hs-opentelemetry-otlp`](../../otlp), which is generated by
[`wireform-proto`](https://github.com/iand675/wireform-). Because
`wireform-core` requires `base >= 4.18`, this package needs GHC 9.6 or newer.

## HTTP/protobuf (default)

The default transport is HTTP/protobuf (`/v1/traces`, `/v1/metrics`,
`/v1/logs`). No extra setup is required beyond the `wireform-proto`
`source-repository-package` already configured in the repository `cabal.project`.

## gRPC (optional)

The `grpc` cabal flag (off by default) enables the native OTLP/gRPC transport in
`OpenTelemetry.Exporter.OTLP.GRPC`, built on
[`wireform-grpc`](https://github.com/iand675/wireform-) (a wireform-native fork
of grapesy) instead of grapesy. The per-service RPC bindings are hand-written
against the generated request/response records.

Enabling it requires the additional wireform packages `wireform-grpc`,
`grpc-spec`, `wireform-http2`, `wireform-network`, and the `http-semantics`
fork. These are intentionally not listed in the repository `cabal.project`'s
`source-repository-package`: at the pinned revision `wireform-grpc.cabal`
references an `extra-source-files` data file that is not tracked in git, which
makes cabal's source-repository-package extraction reject it. To build with
`-fgrpc`, add those subdirs as local packages (a working checkout where the
data file is present, or once the upstream packaging is fixed):

```cabal
-- cabal.project.local
packages:
  /path/to/wireform-/wireform-network
  /path/to/wireform-/wireform-http2
  /path/to/wireform-/http-semantics
  /path/to/wireform-/grpc-spec
  /path/to/wireform-/wireform-grpc

package grpc-spec
  flags: -snappy
```

`wireform-network` links against OpenSSL, so `pkg-config` and the OpenSSL
development headers must be installed.
